Method

How it works

The product is not a black box that invents directions. It is a graph-native ranking system that surfaces underconnected ideas and shows why they deserve attention.
1. Build the graph Papers are converted into directed causal-claim edges. This is the only step where AI is allowed.
2. Score missing edges The engine combines gap signals, path support, motif completion, hub penalties, and field distance.
3. Rank what to do next Users can re-rank for frontier bets, tractable fast-follow work, underexplored areas, or bridge ideas.
Use cases

Who this is for

The beta is designed for economists, funders, editors, and research organizations that want a structured way to inspect what the field is neglecting.

Researchers

Spot tractable papers Start from strong graph evidence and convert latent links into empirical designs.
Search for bridge topics Find pairs of concepts that connect separate literatures with minimal prior contact.

Funders and labs

Build portfolios Compare underexplored, boundary, and fast-follow projects instead of relying on vague novelty rhetoric.
Interrogate the field map See which target areas and source-to-target corridors are rising inside the graph.

Editors and reviewers

Contextualize submissions Understand whether a paper sits on a crowded path or opens a low-contact route.
Surface neglected combinations Review research agendas through the lens of graph structure rather than field prestige alone.
Launch plan

Public beta architecture

The launch path keeps the stack lean: a static public website, a public app, and a read-only mounted economics database behind it.

Website

frontiergraph.com Hosted on Cloudflare Pages. This is the public face of the project and should stay fast, static, and cheap.
your-service.run.app Hosted on Google Cloud Run for the first beta. Move it behind app.frontiergraph.com once you want a branded production URL.

Data posture

Read-only beta The first public version serves a mounted SQLite build of the economics corpus. No user accounts or mutable state yet.
Later upgrade path Once the product needs feedback, accounts, or saved research agendas, the storage layer can move to PostgreSQL.

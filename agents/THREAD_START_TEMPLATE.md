# Thread Start Template

Use a new thread message along these lines:

```text
Please resume FrontierGraph from the current canonical handoff pack in:

- agents/LATEST.md
- agents/FULL_HANDOFF_2026-04-14.md
- agents/NEXT_STEPS.md

Treat those files as the source of truth.

Current branch policy:
- main = stable public line
- v3_paper = next research cycle
- codex-post-seminar-cleanup-2026-04-13 = archive only

Before changing anything, summarize:
1. where the project is,
2. which branch should be used for this task,
3. what the immediate next task is,
4. what files you will inspect or change.
```

## If The Task Is More Specific
Append one line like:
- "Focus on v3_paper research work only."
- "Focus on main-branch maintenance only."
- "Focus on the paper and deck only."
- "Focus on website/downloads maintenance only."

- This template should stop a new thread from defaulting to stale product-state assumptions.

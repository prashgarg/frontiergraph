# Ontology v2 Plan (REVISED — based on actual data exploration)

## Goal

Revamp the concept ontology from ~6,752 broad concepts to a finer-grained, hierarchical vocabulary that:
1. Has enough specificity that author overlap is no longer saturated
2. Supports "don't suggest within same family" filtering
3. Is grounded in external sources (Wikidata, JEL classification) rather than purely corpus-derived
4. Enables a multi-level benchmark (broad themes → specific research questions)

## Branch strategy

- `paper-incubation` — current work, frozen, do not modify
- `paper-incubation-v2ontology` — new branch for this work
- `main` — public, untouched

## Four seed sources for the new ontology

### Source 1: JEL Classification (structured, economics-specific)

The AEA JEL guide (https://www.aeaweb.org/jel/guide/jel.php) provides:
- Level 1: 20 broad categories (A-Z minus some)
- Level 2: ~100 subcategories (e.g., P14 Property Rights)
- Level 3: ~400 detailed codes (e.g., P140 Capitalist Systems: Property Rights)
- Level 4 (NEW, we create): keywords within each code (e.g., "Enforcement of Rights", "Private Property")
- Level 5 (NEW, LLM-expanded): seed terms generated by an LLM within each L4 keyword

This gives a hierarchical backbone: JEL L1 → L2 → L3 → L4 keywords → L5 seed terms.

### Source 2: Wikidata concepts (broad, grounded, external)

Wikidata has structured economics-relevant concepts with:
- Unique identifiers (Q-IDs)
- "Instance of" / "subclass of" hierarchies
- Multilingual labels
- Cross-references to other knowledge bases

Advantages:
- Brings in concepts that may not appear in the corpus yet (emerging topics)
- Provides canonical names and aliases
- Has hierarchical structure built-in
- Covers both economics and adjacent fields (political science, sociology, public health)

### Source 3: OpenAlex concepts/topics + n-grams

OpenAlex provides:
- **Topics**: 4,500+ hierarchical topics assigned to each work (via their topic model)
- **Concepts** (legacy, being deprecated but still available): hierarchical concept tree
- **N-grams**: OpenAlex has an n-grams endpoint — for each work ID, you can retrieve the most frequent 1-grams, 2-grams, and 3-grams from the full text (via the `ngrams` endpoint: `GET /works/{id}/ngrams`)
- **Keywords**: OpenAlex also stores author-assigned keywords per work

The n-grams endpoint is particularly valuable because it gives us *paper-level term frequency* without needing full text — we can identify which specific terms each paper uses most, creating a much finer-grained concept vocabulary.

### Source 4: Corpus-derived labels (current approach, refined)

The current 6,752 concepts come from LLM extraction + normalization. These can be:
- Mapped to JEL codes and Wikidata concepts
- Split using context metadata (country, year, method)
- Used as a bridge between the external sources and the paper-specific language

## Embedding strategy for ontology construction

### Current approach
The current ontology uses text embeddings for soft matching in the tail recovery stage — after deterministic lexical matching fails, embeddings rank plausible concept matches.

### What's better than text-embedding-3-large (as of April 2026)?

| Model | Dimensions | Key strength | Notes |
|-------|-----------|-------------|-------|
| text-embedding-3-large (OpenAI) | 3072 | General-purpose, good baseline | What we currently use |
| Voyage-3-large (Voyage AI) | 1024 | Strongest on retrieval benchmarks | Often beats OpenAI on MTEB |
| Cohere embed-v4 | 1024 | Multilingual, compressed | Good for clustering |
| NV-Embed-v2 (NVIDIA) | 4096 | Highest MTEB scores as of 2025 | Very large, expensive |
| Jina embeddings v3 | 1024 | Good for academic/scientific text | Task-specific adapters |
| BGE-M3 (BAAI) | 1024 | Multi-granularity (dense + sparse + colbert) | Open-source, free |
| GTE-Qwen2 | 1536-8192 | Alibaba, strong on Chinese + English | Open-source |
| E5-Mistral-7B-instruct | 4096 | Instruction-tuned LLM embeddings | High quality, expensive |
| SciBERT / SPECTER2 | 768 | Trained on scientific text specifically | Free, domain-specific |

### Ensemble approach for ontology construction

Rather than relying on one embedding model, we could:

1. **Multi-model voting**: Embed each concept label with 3+ models, compute similarity matrices for each, and accept a merge only if 2+ models agree it's a match. This prevents any single model's biases from dominating.

2. **Dense + sparse hybrid**: Use a dense embedding (Voyage or OpenAI) for semantic similarity AND a sparse representation (BM25 or tf-idf on concept label tokens) for lexical matching. A merge requires both signals to agree.

3. **Hierarchical embedding**: First embed at the JEL-code level to establish broad clusters, then embed within each cluster at finer granularity. This naturally creates the hierarchy: concepts that are close within the same JEL cluster are siblings, concepts in different clusters are in different families.

4. **LLM-as-judge adjudication**: For ambiguous cases where embedding models disagree, use an LLM to adjudicate: "Should 'carbon emissions' and 'CO2 emissions' be the same concept or different concepts in an economics research graph?" This replaces the current hard/soft mapping with a more principled decision.

## Hierarchical structure

The final ontology would have 4-5 levels:

```
Level 0: Domain (Economics, Finance, Adjacent)
Level 1: JEL category (e.g., Q - Agricultural and Natural Resource Economics)
Level 2: JEL subcategory (e.g., Q5 - Environmental Economics)
Level 3: Specific concept (e.g., "carbon pricing")
Level 4: Context-specific variant (e.g., "carbon pricing [EU ETS]")
```

This enables:
- "Don't suggest within same L2 family" filtering
- Family-aware comparison (siblings at L3)
- Gap/boundary distinction grounded in JEL hierarchy (gap = same L2, boundary = different L2)
- Author-expertise at different granularities

## OpenAlex n-grams

The OpenAlex n-grams endpoint (`/works/{id}/ngrams`) returns:
- Up to 10,000 n-grams per work
- Each n-gram has: ngram text, ngram count, ngram tokens, term frequency
- Available for works with full text in OpenAlex (substantial coverage for recent papers)
- Endpoint: `GET https://api.openalex.org/works/W2023271753/ngrams`

This can be used to:
1. Build a corpus-grounded term frequency inventory
2. Identify which specific terms (not just extracted concepts) each paper uses
3. Create a finer-grained vocabulary from the most distinctive 2-grams and 3-grams
4. Weight concept importance by actual paper-level term frequency

## Implementation plan

### Phase 1: Gather seed vocabularies
- Scrape JEL guide (all levels + keywords + examples)
- Query Wikidata for economics-relevant concepts
- Fetch OpenAlex topics + n-grams for a sample of papers
- Export current 6,752 concepts with their labels and support counts

### Phase 2: Build hierarchical backbone
- Map JEL codes to Wikidata concepts (many will have direct Q-ID links)
- Use LLM to expand JEL keywords into L4/L5 seed terms
- Create the hierarchy: JEL L1 → L2 → L3 → expanded terms

### Phase 3: Map corpus concepts to the hierarchy
- Embed all concept labels + JEL terms + Wikidata labels
- Use multi-model voting to assign each corpus concept to the hierarchy
- Identify concepts that don't map (genuinely new economics vocabulary)
- Split broad concepts using context metadata where warranted

### Phase 4: Build the new graph
- Re-run the normalization pipeline with the finer-grained ontology
- Compare graph statistics (concept count, edge density, degree distribution)
- Re-run the benchmark on the new graph
- Test whether author overlap is now sparser

### Phase 5: Evaluate
- Compare old vs new ontology on the benchmark
- Test whether the finer granularity recovers the Sourati & Evans effect
- Test whether the hierarchy enables better family-aware filtering
